# Clarity over cleverness

Atlas interfaces are used while people make time-sensitive business decisions. UI should make the next action obvious and describe the object affected by that action.

## Do

- Name the object in destructive or financial actions.
- Put status and consequence near the control that changes state.
- Use `component:alert` for critical warnings before a destructive action.

## Do not

- Do not hide consequences behind generic labels.
- Do not use playful copy for risk, money, billing, or access decisions.
